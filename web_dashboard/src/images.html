Candidate paths

WebRTC video stream Best fit for high FPS and low latency after encoding H.264/VP8 on the backend.
CompressedImage snapshots Fastest to implement with current ROS topics, but browser refresh and JPEG decode can become the bottleneck.
WebSocket binary frames Good control over backpressure; pair with createImageBitmap or OffscreenCanvas to keep the UI responsive.
WebCodecs pipeline Most useful once we send encoded video chunks instead of JPEG snapshots; it gives low-level frame control and can run decode work away from the main UI path.
